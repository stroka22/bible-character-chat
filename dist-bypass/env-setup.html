Bible Character Chat - Environment Setup

Follow these steps to fix the "Something went wrong" error by properly configuring your environment variables.

Step 1: Create/Edit the .env file

Create a file named .env in the root directory of your project with the following content:

# Bible Character Chat - Environment Variables

# OpenAI API Configuration (required for chat functionality)
VITE_OPENAI_API_KEY=sk_your_openai_api_key_here

# Stripe Configuration (required for subscription features)
VITE_STRIPE_PUBLIC_KEY=pk_test_your_stripe_publishable_key_here
VITE_STRIPE_PRICE_MONTHLY=price_your_monthly_price_id_here
VITE_STRIPE_PRICE_YEARLY=price_your_yearly_price_id_here

# Supabase Configuration (optional but recommended)
VITE_SUPABASE_PROJECT_REF=your_project_ref_here
VITE_SUPABASE_EDGE_FUNCTION_URL=https://your_project_ref.functions.supabase.co/create-checkout-session

# Optional: Enable verbose logging (true/false)
VITE_DEBUG_MODE=true

Important: Replace the placeholder values with your actual API keys and IDs.

For OpenAI, get your API key from https://platform.openai.com/account/api-keys

For Stripe, get your publishable key from https://dashboard.stripe.com/apikeys

Security Warning: For Stripe, always use a publishable key (starts with pk_test_ or pk_live_), never a secret key (starts with sk_).

Step 2: Stop any running servers

Run this command to stop any servers running on the common ports:

lsof -ti:5173,5174,5175,5179,5180 | xargs kill -9

Step 3: Clean build artifacts

Remove old build files to ensure a clean build:

rm -rf dist
rm -rf node_modules/.vite

Step 4: Rebuild the application

Build the application with the new environment variables:

npm run build

Step 5: Start the server

Start the server to serve the application:

npx http-server dist -p 5179

Success! After following these steps, your application should work correctly without the "Something went wrong" error.

Still having issues?

If you're still experiencing problems, try these additional troubleshooting steps:

  1. Check that your API keys are valid and active
  2. Clear your browser cache using our Cache Cleaner tool
  3. Try a different browser or an incognito/private window
  4. Ensure the service files are using the correct export format (ESM exports)
